### What we fixed this audit

- SignInDialog Tab focus-trap (was missing).

### What's still open

- AuthChip menu doesn't respond to ↑/↓ arrow keys to move between items.
  This is a "menu" role best-practice; with only 2 items it's not a
  blocker. **Defer until we add a 3rd menu item.**
- SignInDialog tabs are role="tab" but the panels aren't role="tabpanel"
  with `aria-controls` linking. **Defer** — the form is the panel and
  screen readers announce it correctly because the tab labels swap the
  form contents; we never have multiple panels alive at once.

## 6. Living-document policy

To keep this from drifting again, when we ship the next significant
feature:

1. If the C4 shape changes (new external service, new container, new top-
   level component) → update `docs/SYSTEM.md` Mermaid blocks **in the same
   commit** as the code.
2. If a load-bearing technical decision was made → add an ADR section to
   `docs/SYSTEM.md`.
3. Always run `npx @mermaid-js/mermaid-cli -i docs/SYSTEM.md` before
   committing — it catches Mermaid syntax breaks before they hit GitHub.
4. `.env.example` and `README.md`'s env table are part of the surface; if
   the env shape changes, update both.

Documented this in AGENTS.md's "When you're done with a task" checklist.
